RightSpend Configuration Parameters

Common Configuration Parameters

Parameter Description Value
AllowedRegions Specifies the list of AWS regions where RightSpend is permitted to modify Convertible Reserved Instances (cRIs).
AllowedAccounts Specifies the list of AWS accounts where RightSpend is permitted to modify cRIs.
AllowedPlatforms Specifies the list of platforms that RightSpend is allowed to cover. “All” by default.
AllowLongDeferral Allows deferral to the latest possible date within the "MaxExchangeDays" period.
AllUpfrontDays Upgrades cRIs to all-upfront when they have fewer than the specified number of days remaining before expiration.
Buffer Specifies the dollar-per-hour amount ($/hr) to leave running as on-demand usage.
MaxExchangeDays Specifies the maximum number of days until expiration for cRIs that RightSpend is allowed to modify.
RecentEC2CutOff Instructs RightSpend to ignore EC2 instances started less than the specified number of minutes ago.
LazyThreshold Allows RightSpend to skip optimization runs for specific hours if the expected savings gain falls below the defined threshold.

Per-Region Configuration Parameters

Parameter Description Value
MaxHourlyCommitment Maximum $/hr of all cRIs in this region across all accounts.
MaxTotalCommitment Total commitment for a region across all accounts.
DecreaseRate Speed of decreasing cRI commitment when overcommitted.
IncreaseRate Speed of increasing cRI commitment when there’s on-demand spend.
ReactToSPDiff Enables sharp changes to commitment during modifications to the structure of Savings Plans (SPs).
